# Cambios – 26 de junio de 2025

## Resumen
Mejoras en filtros dinámicos, ranking de móviles, reubicación de UI, y corrección de bugs en detalle de móvil.

---

## 1. Filtro dinámico de Tipo de Servicio ✅
**Archivos modificados:**
- `components/map/MovilesZonasLayer.tsx`
- `components/map/MapView.tsx`
- `app/dashboard/page.tsx`

**Descripción:**
El combo de "Tipo de Servicio" en la capa de Móviles/Zonas ahora se carga dinámicamente desde el campo `servicio_nombre` de la tabla de pedidos y services, sin valores repetidos. Antes estaba hardcodeado con opciones "Pedidos" y "Services".

**Cambios técnicos:**
- `MovilesZonasServiceFilter` cambiado de union type (`'all' | 'pedidos' | 'services'`) a `string`
- Eliminada función `classifyService()` que hacía match estático
- Nuevo prop `tiposServicioDisponibles: string[]` en `MovilesZonasLayer` y `MapView`
- `page.tsx` deriva los tipos de servicio via `useEffect` desde `pedidosCompletos` + `servicesCompletos` → campo `servicio_nombre`
- Filtro compara `tipo_de_servicio` directamente (case-insensitive)

## 2. Ranking de Móviles – Nuevas columnas ✅
**Archivo modificado:** `components/ui/LeaderboardModal.tsx`

**Descripción:**
Se reestructuró completamente el modal de ranking con las siguientes columnas:

| Columna | Lógica |
|---------|--------|
| Móvil | Nombre con ranking y color |
| Atrasados | Pedidos con `delay < 0` y `estado_nro = 1` |
| Pendientes | Pedidos con `estado_nro = 1` |
| No Entregados | `estado_nro = 2` y `sub_estado_desc ≠ '3'` |
| Entregados | `estado_nro = 2` y `sub_estado_desc = '3'` |
| % Cumplimiento | `entregados / (entregados + pendientes)` |
| % Cumplimiento en Hora | `entregados con delay ≥ 0 / total entregados` |

**Cambios técnicos:**
- Import de `computeDelayMinutes` desde `@/utils/pedidoDelay`
- SortKey ampliado a 6 opciones
- Resumen con 3 cards (Móviles, Entregados, Cumplimiento)
- 6 botones de ordenamiento
- Tabla con barras de progreso para % Cumplimiento y % En Hora
- Modal ampliado a `max-w-4xl`

## 4. Botón "Pendientes" en detalle de móvil – Corregido ✅
**Archivo modificado:** `app/dashboard/page.tsx`

**Descripción:**
Al hacer click en "Pendientes" desde el popup de detalle de un móvil, el botón no producía ningún efecto visible porque `showPendientes` ya era `true` desde la carga inicial.

**Corrección:**
El handler `handleShowPendientes` ahora además de mostrar marcadores, abre la tabla de Pedidos (`setIsPedidosTableOpen(true)`), dando al usuario acceso inmediato a la vista detallada de los pedidos pendientes.

## 5. Id Terminal mostraba N/A – Corregido ✅
**Archivos modificados:**
- `app/api/movil-session/[id]/route.ts`
- `components/map/MovilInfoPopup.tsx`

**Descripción:**
El campo "Id Terminal" en el popup de móvil siempre mostraba "N/A" por dos causas:

1. **Bug PascalCase en API**: La API externa de Riogas devuelve `data.IdTerminal` (PascalCase), pero el proxy leía `data.idTerminal` (camelCase).
2. **Fuente incorrecta en UI**: El popup leía `movil.terminalId` (nunca poblado) en vez de `sessionData.idTerminal` (datos de sesión ya cargados).

**Corrección:**
- API ahora lee `data.IdTerminal || data.idTerminal` (compatible con ambas formas)
- Popup ahora usa `sessionData?.idTerminal || movil.terminalId || 'N/A'`
